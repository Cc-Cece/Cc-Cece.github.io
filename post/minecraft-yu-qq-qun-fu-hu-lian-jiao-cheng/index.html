Minecraft与QQ群服互联教程

2025-07-15 4 min read

一、介绍

  • 还在羡慕 Discord 和 Telegram 的机器人么，还在为需要魔法才能使用而头疼么，现在，基于 Forge和 Fabric 的群服连接机器人来了。
  • 此模组区别于插件版的群服连接，不想折腾插件的服主可以试试。

二、提示

该模组不止一个教程，本教程适用于Lagrange协议端，若您在寻找其他协议端的教程，您可以略过本教程。

三、准备工作

  • 一个QQ账户（建议使用价值不大的小号）
  • 适用与您的设备的Lagrange版本
  • 适用于您的Minecraft版本的模组文件
展开版本选择提示

四、教程开始

1.配置Lagrange

Windows Linux/MacOS
双击 Lagrange.OneBot.exe 运行即可 依次运行以下指令： chmod +x ./Lagrange.OneBot ./Lagrange.OneBot

第一次运行时, 会在同级目录下自动生成默认的 appsettings.json 配置文件, 你需要正确修改配置文件以设置 Lagrange

在配置文件按需修改后（推荐使用扫码登录）, 在命令行中按任意键, Lagrange 将正式运行在同一文件夹下会出现一张登录二维码图片 qr-0.png, 在二维码过期前尽快使用手机 QQ 扫码连接

展开修改配置文件提示

请仔细阅读注释和高亮内容

{
  "Logging": {
    "LogLevel": {
      "Default": "Information",  // 提 Issue 时请切换到 Trace
      "Microsoft": "Warning",
      "Microsoft.Hosting.Lifetime": "Information",
    },
  },
  "SignServerUrl": "https://sign.lagrangecore.org/api/sign",
  "MusicSignServerUrl": "",
  "Account": {
    "Uin": 0,  // Uin 填写 0 以使用扫码连接
    "Password": "",  // 不填写密码以使用扫码连接
    "Protocol": "Linux",  // 使用 Linux 协议
    "AutoReconnect": true,
    "GetOptimumServer": true,
  },
  "Message": {
    "IgnoreSelf": true,  // 忽略 Bot 自身的消息
    "StringPost": false,
  },
  "QrCode": {
    "ConsoleCompatibilityMode": false,
  },
  "Implementations": [  // 服务实现 支持多链接
    {
      "Type": "ReverseWebSocket",
      "Host": "127.0.0.1",
      "Port": 8080,
      "Suffix": "/onebot/v11/ws",
      "ReconnectInterval": 5000,
      "HeartBeatInterval": 5000,
      "HeartBeatEnable": true,
      "AccessToken": "",
    },
    {
      "Type": "ForwardWebSocket",
      "Host": "127.0.0.1",
      "Port": 8081,//设定您已开放的端口（TCP和UDP协议均需打开）
      "HeartBeatInterval": 5000,
      "HeartBeatEnable": true,
      "AccessToken": "",
    },//如果您的配置文件没有这一段请手动添加
  ],
}

注意：以 // 开头的为注释, 试图复制粘贴到实际的配置文件中时务必删除

2.配置模组

1.首先请将mod文件移至服务器的mods文件夹（运行环境: 客户端无效, 服务端需装）

2.第一次开启您的服务器，让模组生成配置文件。然后关闭服务器。

3.打开服务器根目录的mcbot文件夹，找到config.toml文件，并打开查看。

4.检查以下项是否正确（其它请参考注释）

  • group-id-list = [12345,22344,66789]# 支持多个q群,用,分隔
  • url = “ws://127.0.0.1:5140” # 地址与端口号，请与您的Lagrange设定值一致：
  • bot-id = 100023 # 机器人qq

5.重新开启服务器，检查是否已经正常运行
