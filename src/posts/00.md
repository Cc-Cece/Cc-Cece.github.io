## 一、介绍

- 还在羡慕 Discord 和 Telegram 的机器人么，还在为需要魔法才能使用而头疼么，现在，基于 Forge和 Fabric 的群服连接机器人来了。
- 此模组区别于插件版的群服连接，不想折腾插件的服主可以试试。



## 二、提示

该模组不止一个教程，本教程适用于[Lagrange](https://link.mcmod.cn/target/aHR0cHM6Ly9naXRodWIuY29tL0xhZ3JhbmdlRGV2L0xhZ3JhbmdlLkNvcmU=)协议端，若您在寻找其他协议端的教程，您可以略过本教程。

## 三、准备工作

- 一个QQ账户（建议使用价值不大的小号）
- 适用与您的设备的[Lagrange版本](https://github.com/LagrangeDev/Lagrange.Core/releases)
- 适用于您的Minecraft版本的[模组文件](https://github.com/Nova-Committee/McBot/releases)

## 四、教程开始

### 1.配置Lagrange

| Windows                           | Linux/MacOS                                                  |
| --------------------------------- | ------------------------------------------------------------ |
| 双击 Lagrange.OneBot.exe 运行即可 | 依次运行以下指令： `chmod +x ./Lagrange.OneBot` `./Lagrange.OneBot` |

### 2.配置模组

1.首先请将mod文件移至服务器的mods文件夹（运行环境: 客户端无效, 服务端需装）

2.第一次开启您的服务器，让模组生成配置文件。然后关闭服务器。

3.打开服务器根目录的mcbot文件夹，找到`config.toml`文件，并打开查看。

4.检查以下项是否正确（其它请参考注释）

- group-id-list = [12345,22344,66789]**# 支持多个q群,用,分隔**
- url = “ws://127.0.0.1:5140” **# 地址与端口号，请与您的Lagrange设定值一致：**![](https://cc.akihito.dpdns.org/post-images/1752585765763.png)
- bot-id = 100023 **# 机器人qq**

5.重新开启服务器，检查是否已经正常运行

出现以下字样即为正常运行：![](https://cc.akihito.dpdns.org/post-images/1752585797555.png)
